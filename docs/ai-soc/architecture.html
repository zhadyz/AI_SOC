THREAT DETECTION PIPELINE

Real-Time Data Flow

End-to-end threat detection and automated response. From network traffic capture to ML-powered analysis to orchestrated incident response.

1

Ingestion

Network traffic captured via Suricata/Zeek. Logs forwarded to Wazuh Manager via Filebeat.

2

Correlation

Wazuh applies correlation rules, enriches events, and generates alerts.

3

ML Analysis

Alert features extracted and sent to ML Inference API for prediction (99.28% accuracy).

4

Triage

Alert Triage Service ranks alerts, queries RAG for context, and prioritizes response.

5

Case Creation

High-priority alerts trigger automatic case creation in TheHive via webhook.

6

Response

Shuffle executes automated playbooks: isolation, blocking, notification, reporting.

SYSTEM INTEGRATIONS

Integration Architecture

Webhook-based event-driven architecture. RESTful APIs, service mesh communication, and asynchronous message passing for loosely coupled services.

SIEM ‚Üí SOAR

Wazuh ‚Üí TheHive

POST /api/alert/create

Wazuh alerts automatically create cases in TheHive. Alert metadata, severity, and observables extracted for investigation.

SOAR ‚Üí AUTOMATION

TheHive ‚Üí Shuffle

POST /api/v1/hooks/{workflow_id}

Case updates trigger Shuffle workflows. Automated playbooks execute response actions based on case status and severity.

AI ‚Üí SIEM

ML Inference ‚Üí Wazuh

POST /predict

Alert features sent to ML API for real-time prediction. Classification results enriched back into Wazuh alerts.

MONITORING ‚Üí ALL

Prometheus Scraping

GET /metrics

13 scrape targets across all stacks. Service health, resource utilization, and custom application metrics.

ANALYSIS ‚Üí ENRICHMENT
